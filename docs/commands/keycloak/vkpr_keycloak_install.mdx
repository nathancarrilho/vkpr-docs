# VKPR keycloak install

## Description

Install Keycloak into cluster. For more information about Keycloak, click [here](https://www.keycloak.org/).

## Commands

Interactive inputs:

```bash
  vkpr keycloak install [flags]
```

Non-interactive without setting values or using `VKPR Values`:

```bash
  vkpr keycloak install --default
```

### Parameters

```txt
  --default           Set all values with default.
  --domain            Define the domain used by the keycloak.          Default: localhost
  --secure            Specifies if the application will have HTTPS.    Default: false
  --admin_user        Define RBAC Super Admin user on the keycloak.    Default: admin
  --admin_password    Define RBAC Super Admin password keycloak.       Default: vkpr123
  --HA                Specifies if the application will have High Availability.   Default: false
  --password          Specifies the Postgresql password.               Default: false
```

:::caution Info
If the VKPR dont identify any postgresql installed in the cluster, he will [install](/docs/commands/postgres/vkpr_postgres_install) to use with Keycloak.
:::

### Setting Postgresql credentials

#### Postgresql

```bash
  rit set credential --provider="postgres" --fields="password" --values="your-password"
```

### Values File Parameters

```jsx title="vkpr.yaml"
global:
  domain:               <String>
  secure:               <Boolean>
  keycloak:
    HA:                 <Boolean>
    ingressClassName:   <String>
    adminUser:          <String>
    adminPassword:      <String>
    metrics:            <Boolean>
    helmArgs:           <Object>
```
