# Ingress

Para que os usuários acessem os aplicativos internos de um cluster K8S, é necessário usar o Ingress. Mas para que o Ingress funcione, você precisará de um Ingress-Controller para gerenciar todo o balanceamento de carga.

O VKPR oferece quatro tipos de Ingress Controller:

- [Consul](https://www.consul.io/docs): O Consul é uma ferramenta multi-rede que resolve os desafios de rede e segurança da operação de microsserviços e infraestrutura em nuvem. Para visualizar os comandos do consul, clique [aqui](/docs/commands/consul/install)

- [nGINX](https://github.com/kubernetes/ingress-nginx): Controlador de entrada para Kubernetes usando NGINX como proxy reverso e balanceador de carga. Para visualizar os comandos do ingress, clique [aqui](/docs/commands/ingress/install)

- [Kong](https://docs.konghq.com/): Gateway de API criado para nuvem híbrida e multinuvem, otimizado para microsserviços e arquiteturas distribuídas. Para visualizar os comandos do kong, clique [aqui](/docs/commands/kong/install)

- [Traefik](https://doc.traefik.io/traefik/): Um proxy reverso HTTP moderno e balanceador de carga feito para implantar microsserviços com facilidade. (Somente em Env local) Para visualizar os comandos do traefik, clique [aqui](/docs/commands/infra/start)
